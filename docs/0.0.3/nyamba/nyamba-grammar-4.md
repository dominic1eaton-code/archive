## I. GRAMMATICAL OVERVIEW

### A. Typological Profile

```yaml
Language Type: Agglutinative-Fusional Hybrid
Word Order: VSO (basic), flexible for pragmatics
Morphological Complexity: High (synthetic)
Head Direction: Mixed (head-final in NPs, head-initial in VPs)
Case System: Postpositional (15 cases)
Number System: Singular-Dual-Plural-Collective (4-way)
Alignment: Nominative-Accusative
Polarity Marking: Grammaticalized symmetric/asymmetric
```

### B. Core Grammatical Categories

```
MAJOR CATEGORIES:
├── Nouns (8 derivational classes)
├── Verbs (12 TAM combinations + polarity)
├── Adjectives (3 derivational types)
├── Adverbs (4 subtypes)
├── Pronouns (9 case forms × 4 numbers × 3 persons)
├── Determiners (demonstratives, quantifiers, articles)
├── Postpositions (15 cases)
├── Conjunctions (coordinating, subordinating)
└── Particles (discourse, modal, polarity)
```

## III. MORPHOLOGICAL SYSTEM

#### Root Types

**1. CVC Roots (Primary)**
```
mun-    'whole, system'
kar-    'structure'
lov-    'close'

```

**2. CV Roots (Reduced)**
```

```

**3. Polonic Roots (Form-preserving)**
```
mu-     wholeness

```

**4. Kolonic Roots (Form-breaking)**
```

| Suffix | Type | Function | Example |

| **-le** | Result/product | Creates outcome | lov-le 'closure (instance)' |
| **-ntu** | Agent/doer | Creates actor | kar-ntu 'structurer' |

| **-nge** | Augmentative | Creates large version | kara-nge 'large structure' |

### D. Mood System (6 moods)

#### D.1 Indicative (Unmarked)

**Function:** Statements of fact

```
Mungon lova
system close.IND
'The system closes'
```

#### D.2 Imperative/Jussive/Command

**Structure:** ROOT + -e

**Functions:**
- Direct command (2nd person)
- Jussive (3rd person)
- Hortative (1st person plural)

**Forms:**

| Person | Form | Function | Example |
|--------|------|----------|---------|
| **2SG** | ROOT-e | Direct command | Love! 'Close!' |
| **2PL** | ROOT-e (+ weya) | Plural command | Love weya! 'Close (you all)!' |
| **1PL** | Duva miya ROOT-e | Hortative | Duva miya love 'Let's close' |
| **3SG/PL** | Duva ye/yeya ROOT-e | Jussive | Duva ye love 'Let him/her close' |

**Negative Imperative:**
```
Pattern: nimu + ROOT-e

Nimu love!
NEG close.IMP
'Don't close!'

Nimu duva-e!
NEG go-IMP
'Don't go!'
```

**Polite/Softened Imperatives:**

| Strategy | Form | Example |
|----------|------|---------|
| Question intonation | ROOT-e? | Love? 'Close?' (polite request) |
| Please particle | ROOT-e nga | Love nga 'Please close' |
| Conditional | Engo we ROOT-nga | Engo we lovnga 'If you would close' |
| Modal softener | We ROOT-ngaa? | We lovngaa? 'Would you close?' |

**Complete Imperative Paradigm:**

```
AFFIRMATIVE:
Love!                   'Close!' (you-sg)
Love wedu!              'Close!' (you-dual)
Love weya!              'Close!' (you-pl)
Duva miya love!         'Let's close!' (we)
Duva ye love!           'Let him/her close!' (jussive)
Duva yeya love!         'Let them close!' (jussive)

NEGATIVE:
Nimu love!              'Don't close!' (you-sg)
Nimu love weya!         'Don't close!' (you-pl)
Nimu duva miya love!    'Let's not close!'
Nimu duva ye love!      'Let him/her not close!'

POLITE:
Love nga?               'Please close?'
We love-ngaa?           'Would you close?'
Engo we lovnga?         'If you would close?'
```

#### D.3 Subjunctive

**Pattern:** engo ... ROOT-a

**Function:** Hypothetical, conditional, wishes

```

if 1SG close.SUBJ system structure.CONT
'If I were to close, the system would be structuring'

Engo ye lovi, kontinuitu karile

'If he/she should close, kontinuity would have structured'
```

#### D.4 Conditional

**Pattern:** engo + VERB (any tense)

```

if 1SG close.COND 3SG structure.CONT
'If I close, he/she is structuring'

Engo we lovile, mi lovnga
if 2SG close.PST.COND 1SG close.FUT
'If you closed, I will close'
```

**Marker:** -le

```

'May the system close!'

'May omega close for you'

```

#### D.6 Potential (Ability)

**Marker:** -nga-a

```

'I can close the system'

'He/she cannot close'

'Can you structure kontinuity?'
```

'We two close each other'

## V. SYNTAX

### A. Basic Word Order

**Default: VSO**

```
Lova     mungon      kontinuitu
close    system      kontinuity
'The system closes kontinuity'
```

**Flexible for Pragmatics:**

```
SVO (Topic):
Mungon   lova        kontinuitu
system   close       kontinuity
'The system (as for it) closes kontinuity'

OSV (Topicalized Object):
Kontinuitu   lova      mungon
kontinuity   close     system
'Kontinuity, the system closes'

VOS (Focus on Object):
Lova     kontinuitu   mungon
close    kontinuity   system
'(It) closes kontinuity, the system (does)'
```

**Template:**
```
(DET) (QUANT) NOUN (ADJ) (REL.CLAUSE) (CASE)

## VI. COMPOSITION & SYNTHESIS

### A. Word Formation Processes

#### A.1 Derivation

**Process: ROOT → AFFIX → DERIVED WORD**

```
EXAMPLE: lov- 'close' → lovntu 'closer'

STEP 1: Identify root
        lov- (closure root)

STEP 2: Select affix
        -ntu (agent nominalizer)

STEP 3: Apply morphophonological rules
        lov + -ntu → lovntu (no changes needed)

STEP 4: Assign meaning
        'one who closes, closer'
```

#### A.2 Compounding

**Pattern: ROOT₁ + ROOT₂ → COMPOUND**

```
EXAMPLE: mun + ka → munka 'system-structure'

TYPE 1: Head-Final
mun-ka              'system-structure' (structure of system)
Ω-sistem            'omega-system'

TYPE 2: Coordinate
P-K                 'polonic-kolonic' (duality)
mu-vu               'form-flux'

TYPE 3: Recursive
Ω-goma-ntu          'omega-system-person'
```

#### A.3 Inflection

**Process: STEM → INFLECTIONAL AFFIX → INFLECTED FORM**

```
EXAMPLE: mungon → mungon-ya 'systems'

STEP 1: Identify stem
        mungon (system)

STEP 2: Select inflection
        -ya (plural marker)

STEP 3: Apply
        mungon + -ya → mungon-ya

STEP 4: Check for agreement
        If formal: all modifiers take -ya
```

### B. Sentence Assembly

**Sentence Building Algorithm:**

```
INPUT: Semantic propositions
OUTPUT: Grammatical Nyamba sentence

STEP 1: PREDICATE SELECTION
  - Choose verb root
  - Apply polarity (mu-/ko- if needed)
  
STEP 2: ARGUMENT STRUCTURE
  - Identify subject, object, obliques
  - Assign case markers
  
STEP 3: TAM MARKING
  - Select tense
  - Select aspect
  - Select mood
  
STEP 4: WORD ORDER
  - Default: VSO
  - Adjust for topic/focus
  
STEP 5: AGREEMENT
  - Number agreement (if formal)
  - Case agreement
  
STEP 6: FINAL ASSEMBLY
  - Insert particles
  - Apply phonological rules
```

**Example:**

```
SEMANTIC: 'The two systems closed kontinuity yesterday'

STEP 1: PREDICATE
  - Root: lov- 'close'
  - Tense: -ile (past)
  - Result: lovile

STEP 2: ARGUMENTS
  - Subject: mungon-du 'two systems'

close.PST system-DU kontinuity yesterday

```

### C. Synthesis Rules

#### C.1 Morphophonological Processes

**Vowel Coalescence:**
```
RULE: V₁ + V₂ → V (merged)

/a/ + /a/ → /aa/ (long)
/e/ + /a/ → /ya/
/o/ + /a/ → /wa/

EXAMPLE:
lova + -a → lova (no change, same vowel)
```

**Nasal Assimilation:**
```
RULE: /n/ assimilates to following consonant

/n/ + /p,b/ → [m]
/n/ + /k,g/ → [ŋ]

EXAMPLE:
mun-pa → mumpa [mum.pa]
mun-ko → muŋko [muŋ.ko]
```

**Consonant Weakening (rapid speech):**
```
RULE
: Intervocalic /t,k/ → tap/fricative

/t/ → [ɾ]
/k/ → [x] or ∅

EXAMPLE:
Careful: mungon [mu.ŋon]
Rapid: mungun [mu.ŋuŋ]
```

#### C.2 Stress Assignment

**RULE: Penultimate stress**

```
Algorithm:
IF word has 1 syllable:
    stress = initial
ELSE IF word has 2+ syllables:
    stress = second-to-last syllable
ELSE IF final syllable has long vowel:
    stress = final

EXAMPLES:
mu.NGO.ntu      (3 syllables → stress syllable 2)
lo.VA.nga       (3 syllables → stress syllable 2)
kon.ti.NU.i.tu  (5 syllables → stress syllable 4)
```

#### C.3 Syllabification

**Algorithm:**

```
INPUT: Phoneme sequence
OUTPUT: Syllabified word

STEP 1: Mark vowels as syllable nuclei
STEP 2: Attach onset consonants (left-to-right)
STEP 3: Attach coda nasals (if any)
STEP 4: Apply maximal onset principle

EXAMPLE: /mungon/
STEP 1: m u ng o n → [u] [o] are nuclei
STEP 2: [mu] [ngo] (attach consonants to following vowel)
STEP 3: [mun] [gon] (attach final /n/ as coda)
OUTPUT: mun.gon
```
