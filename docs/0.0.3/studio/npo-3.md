# 📌 STUDIO CLASS MODEL

**Studios are first-class reusable entities.**
They are **renewable, repurposable, scalable, and adaptable**.
They intersect with ventures through **capability classes**, each of which encodes a *domain + execution capability + venture outcome type*.

---

## I. STUDIO CLASS DEFINITION

A **Studio Class** is a *semantic type* attached to a StudioCo that encodes:

1. **Domain / vertical** — e.g., mobility, fintech, health, community programs
2. **Capability stack** — e.g., software engineering, strategy, marketing
3. **Output category** — the kinds of ventures it seeds or supports
4. **Reusable assets** — tooling, knowledge, IP, playbooks

Each StudioCo can hold **multiple classes** (a taxonomy) and may evolve over time.

---

## II. STUDIO CO CLASS SCHEMA

Here’s a reusable schema for a studio class:

```
    ]

        ip_modules,
        tooling,
        playbooks,
        labs
    ]
    repurposeRules: { ... }  // how it transitions to other classes

}
```
