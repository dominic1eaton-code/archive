## II. ARCHIVE LIFECYCLE MODEL (HOW OBJECTS EVOLVE)

Every object **must** be in exactly one lifecycle state.

```
[Capture] ‚Üí [Formed] ‚Üí [Validated] ‚Üí [Active] ‚Üí [Canonical] ‚Üí [Archived]
                       ‚Üò [Rejected]     ‚Üò [Deprecated]
```

### **State Definitions**

#### **1. Capture**

* Raw, unstructured
* No authority yet
* Immutable timestamp

Examples:

* Notes
* Brain dumps
* Meeting scribbles

---

#### **2. Formed**

* Structured
* Tagged
* Linked to origin

Examples:

* Draft docs
* Structured notes
* Early diagrams

---

#### **3. Validated**

* Reviewed
* Authority assigned
* Constraint-checked (OVCS if applicable)

Examples:

* Approved strategies
* Signed decisions
* Cleared frameworks

---

#### **4. Active**

* Currently governing or used
* Subject to versioning
* Referenced by execution systems

---

#### **5. Canonical**

* Highest authority
* Non-contradictable without formal override
* Minimal, stable, protected

Examples:

* Msingi doctrine
* Core constraints
* Kernel strategies

---

#### **6. Archived**

* No longer active
* Retained for lineage, audit, learning

---

#### **7. Deprecated**

* Still referenced historically
* Explicitly superseded
* Not valid for new use

---

#### **8. Rejected**

* Invalid
* Failed validation
* Retained only for learning
